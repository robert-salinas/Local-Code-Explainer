# Guía de Contribución

¡Gracias por querer mejorar Local Code Explainer!

## Cómo contribuir

1. **Fork** del repositorio.
2. Crea una **rama** para tu funcionalidad (`git checkout -b feature/nueva-mejora`).
3. Haz tus **commits** siguiendo convenciones claras.
4. Asegúrate de que los **tests** pasen (`pytest`).
5. Abre un **Pull Request**.

## Estilo de Código
- Seguimos PEP 8 para Python.
- Usamos Type Hints siempre que sea posible.
- Documentamos las funciones principales.

## Reporte de Errores
Si encuentras un bug, abre un Issue describiendo:
- Tu sistema operativo.
- El modelo de Ollama que usas.
- El archivo de código que causó el problema.
